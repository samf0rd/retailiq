mart_revenue

Daily/monthly gross revenue with month-over-month growth via window functions.

raw.orders→ stg_orders→ int_orders_enriched→ mart_revenue 
with monthly as (
  select
    date_trunc('month', order_purchase_ts) as order_month,
    sum(item_total) as gross_revenue
  from int_orders_enriched
  group by 1
)
select
  order_month,
  gross_revenue,
  gross_revenue - lag(gross_revenue) over (order by order_month)
    as mom_abs,
  round(100.0 * (gross_revenue / lag(gross_revenue)
    over (order by order_month) - 1), 2) as mom_pct
from monthly
order by order_month;
↑ Window function — LAG() OVER (ORDER BY month) computes period-over-period without a self-join. The SQL-showcase construct for this mart.
Tests on this model
  • ✓ not_null(order_month)
  • ✓ unique(order_month)
  • ✓ custom: gross_revenue >= 0
